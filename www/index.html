RJack
Java component gems for JRuby
Project | Download

Summary

RJack provides broadly used open source java components as gems for use with JRuby. Each gem may be used simply and solely to deliver java package dependencies to a consuming ruby/java application. However, most of these gems also provide either setup facades or adapters for direct use from ruby code. RJack helps bring the advantages of RubyGems to existing Java applications, and lowers barriers for these applications to take advantage of better ruby facilities in such areas as file system and command line interaction, Ruby or DSLs as better "configuration", etc.

In general, rjack gems closely track upstream java project releases. Gem versions are based on the java project version plus an additional gem release number.

See also relevant usage examples and news in the void * blog.

Gems

Build

  • rjack-tarpit

TarPit provides the glue for an unholy union of build tools, namely Rake with Hoe, and Maven. Its used for building all the rjack gems, as well as other java-in-gem projects.

Logging

  • rjack-slf4j
  • rjack-logback

Gems for SLF4J and Logback provide a complete toolkit for unifying and controlling logging in a heterogeneous application containing ruby and java components. The slf4j gem may simply be used to load SLF4J jars that a java component is dependent on, but also offers a ruby core Logger compatible logging adapter. The logback gem provides a facade for programmatic logback configuration in ruby. Any rjack gem java dependencies on Commons Logging (JCL) are satified via require 'rjack-slf4j/jcl-over-slf4j'.

HTTP Server

  • rjack-jetty
  • rjack-jetty-jsp

Gem packaging of the Jetty Web Server. The jetty gem includes an extensible ServerFactory for Server setup in ruby. The jetty-jsp companion gem adds JSP support when needed.

HTTP Client

  • rjack-httpclient (4.x)
  • rjack-httpclient (3.x)

Gem packaging of the HttpComponents (formerly Jakarta Commons) HttpClient. Includes a ruby setup facade. As dependent project migration to httpclient 4.x is likely to take some time, both 3.x and 4.x branches will be maintained in rjack.

Miscellaneous

  • rjack-commons-codec

Additional gems to satisfy common java dependencies.
